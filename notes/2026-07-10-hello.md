# Starting a dev log

This is a dev log, not a blog. There is no schedule -- entries show up when
there is something worth writing down: a validation result, a design
decision, a methods change and the reasoning behind it. Some months that is
nothing. That is fine.

The rest of this site is a faithful account of what MESA's code does today.
This section is the opposite kind of writing: it is allowed to be about
process, dead ends, and things that changed between one version and the
next.

## Why it uses plain Markdown

No CMS, no build step, no database. A note is a Markdown file in `notes/`,
plus one line in a manifest. `git push`, and it is live. That constraint is
deliberate -- the same reason the rest of spatialepi.org is static HTML with
no framework.

Markdown here also renders math, since some notes will need it. Inline like
$z \sim N(0,1)$, or as a block:

$$ p = \frac{1 + \#\{\, T_{\text{perm}} \ge T_{\text{obs}} \,\}}{1 + \text{replications}} $$

That is the Monte-Carlo p-value from the [Bernoulli scan
methodology](/methods-scan.html#mc) page, just here as a rendering test
rather than a real derivation.

## What will actually show up here

Mostly notes tied to real changes: why a default threshold moved, what a
validation run against a new dataset found, a methods decision that did not
fit cleanly into the methods pages' documentation-of-what-is-shipped format.
