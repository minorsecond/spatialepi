Public-health GIS · Offline by design

Geocode addresses and find spatial clusters — without your data ever leaving the machine

MESA is a desktop application for epidemiologists and health-department staff. It cleans and joins your exported tables, attaches latitude/longitude to addresses completely offline, and runs rigorous spatial cluster detection on the results.

Built for PHI-safe fieldwork

Everything runs on your computer against a local copy of authoritative address data — no cloud services, no API keys, no internet connection required.

Offline geocoding

Matches addresses against a local copy of Utah's authoritative address points (UGRC) stored in SQLite, with optional supplemental files for extra coverage.

Spatial cluster detection

Global and local autocorrelation (Moran's I, Getis-Ord Gi*) and a Bernoulli spatial scan — with Monte-Carlo inference that is bit-reproducible across platforms.

Table cleaning & joins

Opens Cognos MHT/MHTML, CSV, and Excel exports. Clean columns, join tables on shared keys, and export to CSV, Excel, or spatial formats.

Standards-aligned

Geocoding validated against a stratified positional harness; spatial statistics cross-checked against reference implementations (esda / spdep).

PHI stays local

No telemetry and no network calls during analysis. Your patient addresses never touch a third-party server.

Cross-platform desktop app

Native builds for Windows, macOS (Apple Silicon), and Linux. Install once, work entirely on your own hardware.

Why offline matters

Public-health data is protected health information. MESA is designed so you can do serious spatial epidemiology without ever exposing it.

  • No data leaves your computer. Geocoding and analysis run entirely against local files.
  • No internet required. Works on an air-gapped machine once the address data is installed.
  • Reproducible science. Deterministic permutation inference produces bit-identical results across Windows, macOS, and Linux for the same input and seed.
  • Auditable. Every export can carry provenance — dataset identity hashes and a record of how the result was produced.
